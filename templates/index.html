Tài liệu API

Upload API cho phép ứng dụng/web khác tải file lên TeleCloud thay mặt bạn, xác thực bằng Bearer token cố định của tài khoản (xem ở mục Cài đặt). Mọi origin đều được phép gọi (CORS mở), giới hạn dung lượng tối đa … MB cho mỗi file.

POST /api/upload-api/upload

Tải lên 1 file. Gửi dạng multipart/form-data, kèm header Authorization: Bearer <token>.

FieldKiểuMô tả
filebắt buộc file Nội dung file cần tải lên.
pathtùy chọn string Thư mục đích, ví dụ /Videos. Mặc định /.
sharetùy chọn string Đặt giá trị public để tự tạo link chia sẻ ngay sau khi upload xong.
uploadIdtùy chọn string ID tự sinh (vd UUID) dùng để theo dõi tiến trình qua SSE ở mục 2 bên dưới.

Phản hồi thành công — 200 OK

{
  "success": true,
  "id": 123,
  "name": "video.mp4",
  "size": 10485760,
  "share_url": "https://host/share/abcdef", // chỉ khi share=public
  "direct_url": "https://host/stream/123" // chỉ khi share=public
}

SSE /api/upload-api/progress/:uploadId?token=...

Theo dõi tiến trình upload theo thời gian thực qua Server-Sent Events. Token truyền qua query string vì EventSource không gửi được header Authorization.

  1. Client tự sinh một uploadId ngẫu nhiên (ví dụ UUID).
  2. Mở kết nối SSE tới endpoint trên trước khi upload, dùng cùng uploadId.
  3. Gửi POST /api/upload-api/upload kèm field uploadId giống bước 1.
  4. Server đẩy SSE event mỗi khi tiến trình thay đổi, kết thúc bằng event done hoặc error. Nhớ gọi .close() sau khi nhận.

Định dạng event SSE

event: telegram
data: {"pct": 42, "message": "Đang gửi lên Telegram..."}

Tên event (event:) chính là stage, có thể là receiving, telegram, done hoặc error. Lắng nghe từng stage bằng addEventListener('receiving', …). Nếu mở SSE mà chưa có session tương ứng, server sẽ đợi tối đa 30 giây rồi gửi event error.
